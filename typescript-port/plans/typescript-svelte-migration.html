LyricDisplay: TypeScript + Svelte 5 Migration

Full-stack migration: React/JS → Svelte 5 + TypeScript + SvelteKit | Electron app for church lyric displays

Overview

Migrate the LyricDisplay Electron desktop app from React 19 / JavaScript to Svelte 5 with runes / TypeScript / SvelteKit. Incremental, file-by-file approach working in the typescript-port/ directory.

// Example: LyricsStore as Svelte 5 class
class LyricsStore {
  lines = $state<LyricsLine[]>([]);
  selectedLine = $state<number>(-1);
  outputSettings = $state({
    output1: { /* ... */ },
    output2: { /* ... */ },
    stage: { /* ... */ }
  });

  get activeLines() {
    return $derived(this.lines.filter(l => l.visible));
  }

  selectLine(index: number) {
    this.selectedLine = index;
  }
}

export const lyricsStore = new LyricsStore();

Risk Mitigation

RiskImpactMitigation
Socket.io + SvelteKit integration High Prototype early in Phase 1. Use hooks.server.ts with raw HTTP upgrade handling.
Electron + SvelteKit dev workflow High Use adapter-node for builds, direct Vite integration for dev. Test hot reload early.
shadcn-svelte component gaps Medium Port missing components manually from shadcn/ui source. Most have 1:1 equivalents.
State persistence (localStorage) Low Svelte stores can persist to localStorage. Port Zustand persist middleware logic.
Native modules (better-sqlite3, keytar) Medium These work in Node.js regardless of frontend framework. No change needed.
